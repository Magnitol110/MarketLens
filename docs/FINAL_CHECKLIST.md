## Потрібні ручні дії

- [ ] **Артем:** у Colab зберегти виконаний `03_final_model.ipynb` у GitHub поверх `notebooks/03_final_model.ipynb`.
- [ ] **Артем:** після цього підтягнути саме оновлений notebook локально.
- [ ] **Артем:** створити Render Blueprint за `render.yaml` і передати Степану URL API.
- [ ] **Степан:** створити Vercel deployment із Root Directory `frontend` та змінною `VITE_API_BASE_URL`.
- [ ] **Артем:** додати Vercel URL у `MARKETLENS_CORS_ORIGINS` на Render і redeploy.
- [ ] **Степан:** створити презентацію, підготувати промову та записати 60–90 секунд резервного демо.
- [ ] **Разом:** відкрити публічний сайт з телефона або іншого ПК.
- [ ] **Разом:** провести три репетиції за фінальним сценарієм Степана.
- [ ] **Артем:** після синхронізації Colab перевірити зміни, зробити один фінальний commit і push.

## Перед виступом

- Відкрити публічний сайт і сторінку MSFT заздалегідь, щоб уникнути cold start.
- Мати локальне демо й відеозапис як два резервні варіанти.
- Не називати сигнал прогнозом гарантованого руху ціни: це відносна до SPY експериментальна класифікація.
- Якщо запитають про точність: macro-F1 `0.339`; baseline `0.281`; dummy `0.169`; якість помірна.
